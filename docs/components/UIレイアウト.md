# components/

このフォルダにはアプリの UI コンポーネントが含まれます。各コンポーネントはプレゼンテーションとユーザー操作を担当し、`context/` や `services/` を通じて状態や外部サービスと連携します。

## 目的

- 画面の見た目とユーザーインタラクションを提供する。
- 共有状態と外部 API のロジックを切り離し、UI 層を簡潔に保つ。
- 再利用しやすく、テストしやすい構造にする。

## 主要コンポーネント

### `ConversationPane`
- 会話表示とメッセージ入力を担うメインコンポーネント。
- ユーザーの英文入力、送信、AI 応答表示、音声入力（STT）を扱う。
- `state.mode` によって表示する履歴を `messages` / `examMessages` で切り替える。
- `useApp()` から `state` と `dispatch` を取得し、`SET_LOADING`、`ADD_MESSAGE` などのアクションを送信する。
- `/api/chat` へのリクエスト送信を行い、返答を受け取って状態に追加する。

### `SupportPane`
- モード切替、ペルソナ設定、例文提案、日本語 Q&A などを提供するサイドサポートウィンドウ。
- `RESET_EXAM` の送信を通じて Exam の再挑戦を管理する。モードはセッション作成時に確定しており、SupportPane からは変更しない。
- `personaInput` の入力と保存を行い、`SET_PERSONA` を dispatch する。
- `support` API への `suggest` / `ask` リクエストを実行して結果を表示する。

### `ExamResult`
- Exam モード終了後にスコアとフィードバックを表示する専用コンポーネント。
- `result: ExamResult` を受け取り、100点満点スコアとフィードバックを表示する。
- `onRetry` コールバックを受け取り、再挑戦アクションをサポートする。

## コンポーネントの設計方針

- 見た目と操作に集中し、状態管理は `context/` に委譲する。
- API 呼び出しやサポート機能のリクエストは必要に応じて `fetch('/api/...')` を行うが、ビジネスロジックは最小限に留める。
- `services/` のラッパーを介して音声入出力を行い、ブラウザ API 依存を隔離する。
- コンポーネント内で副作用を扱う場合は、`useEffect` などの React フックに限定する。

## 画面構成

- `ConversationPane` は主要領域を使ってメッセージ一覧と入力欄を表示する。
- `SupportPane` は右側固定幅のサイドバーとして、補助機能に特化した UI を提供する。
- `ExamResult` は試験結果表示専用で、Exam モード終了後にモーダルやページ内に表示できる。

## UI/UX 要件

- ユーザーと AI の発話を視覚的に区別する（色分け、位置揃えなど）。
- メッセージ送信時は Enter、Shift+Enter で改行を使い分ける。
- 音声認識中は録音状態を明示し、録音停止ボタンの視認性を確保する。
- `SupportPane` のサポート機能はボタン操作と結果表示をシンプルに保つ。
- `ExamResult` はスコアとフィードバックを明瞭に表示し、再挑戦の導線を用意する。

## テスト

- `ConversationPane` の送信フロー、メッセージ表示、音声入力ボタンの挙動をテストする。
- `SupportPane` のモード切替、ペルソナ保存、サポート API 呼び出し、質問応答の表示をテストする。
- `ExamResult` のスコア表示と再挑戦ハンドリングをテストする。

## 拡張

- `ConversationPane` に対してメッセージ編集、削除、翻訳機能、ストリーミング表示などを追加可能。
- `SupportPane` に学習履歴、例文カテゴリ、単語帳機能を追加して学習支援を拡張。
- `ExamResult` に詳細評価基準や改善ポイントの表示を追加する。

以上が `components/` フォルダの設計と要件です。
