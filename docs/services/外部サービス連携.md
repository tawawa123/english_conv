# services/

このフォルダは、外部サービスやブラウザ API との通信を抽象化するユーティリティを収めます。
アプリケーションの UI 層（`components/` や `pages/`）から呼び出され、API キーやリトライ等の環境依存ロジックを隔離します。

## 目的

- STT（音声認識）・TTS（音声合成）・AI クライアント等、外部サービスを呼び出す責務を集中させる。
- コンポーネント側に副作用や設定の詳細を漏らさず、テストしやすいインターフェースを提供する。
- エラー処理、ログ記録、API キー管理などを一元化する。

## 実装済みファイル

### `stt.ts` — 音声認識（Web Speech API）

ブラウザの `SpeechRecognition`（Chrome）/ `webkitSpeechRecognition` をラップする。

**公開 API:**

```ts
sttStart(callbacks: {
  onResult: (transcript: string) => void;
  onEnd?: () => void;
  onError?: (message: string) => void;
}): void

sttStop(): void

isSttSupported(): boolean
```

- 未対応ブラウザでは `onError` に日本語メッセージを渡す。`isSttSupported()` で事前チェック可能。
- マイク権限拒否・無音・ネットワークエラー等を日本語メッセージにマッピング。
- `sttStop()` で現在のセッションを中断し再入可能。

### `tts.ts` — 音声合成（Web SpeechSynthesis）

ブラウザの `SpeechSynthesis` をラップする。

**公開 API:**

```ts
ttsSpeak(text: string, opts?: {
  lang?: string;   // デフォルト "en-US"
  rate?: number;   // デフォルト 1.0
  pitch?: number;  // デフォルト 1.0
  volume?: number; // デフォルト 1.0
}): Promise<void>

ttsStop(): void

isTtsSupported(): boolean

isTtsSpeaking(): boolean
```

- SSR 安全（`typeof window === 'undefined'` でガード）。
- 初回呼び出し時に `voiceschanged` イベントを待ってボイスをロード。言語に最適なボイスを自動選択。
- Chrome の STT 終了直後の TTS 無音バグを回避するため、`cancel()` 後 150ms 待機してから `speak()` を実行。
- `ttsStop()` はクリック再生前の停止にも使用（AI バブルクリック時に現在の読み上げを中断してから再生）。

### `aiClient.ts` — AI クライアント（Gemini API）

Google Gemini API（`@google/genai`）を使って会話・サポートリクエストを処理する。

**使用モデル:** `gemini-3-flash-preview`

**公開 API:**

```ts
sendChat(
  messages: ApiMessage[],
  personaPrompt?: string
): Promise<{ ok: true; reply: string } | { ok: false; error: string }>

sendSupport(prompt: string): Promise<{ ok: true; reply: string } | { ok: false; error: string }>
```

- `sendChat` はチャット形式（`ai.chats.create` + `history`）で送信。最後のユーザーメッセージを `sendMessage` で送る。
- システム命令には以下の制約を常時付与：
  - 英語のみで回答（ペルソナプロンプトの言語に関わらず）
  - 最大 2〜3 文
- `sendSupport` は単発のプロンプト生成（`ai.models.generateContent`）に使用。

## 環境変数

```
GEMINI_API_KEY=...   # Gemini API キー（.env.local で管理、リポジトリ非公開）
```

## セキュリティとプライバシー

- `GEMINI_API_KEY` はサーバーサイド（`pages/api/`）でのみ参照し、クライアントバンドルに含めない。
- `NEXT_PUBLIC_` プレフィックスは使用しない。

## 将来的な拡張

- 外部音声サービス（Google Cloud TTS, Azure 等）の追加対応。
- ストリーミング応答（逐次表示）のサポート（Gemini API は対応済み）。
- セッション永続化サービス（`sessionService.ts`）の追加。

以上が `services/` フォルダの設計要件です。
