# types/

このフォルダは、アプリ全体で共通して使う TypeScript の型定義をまとめています。

## 目的

- 状態管理、API 呼び出し、コンポーネント間のデータ受け渡しで一貫した型安全性を確保する。
- `message` や `exam` などのドメインモデルを明確に定義し、仕様変更時に影響範囲を可視化する。
- 型定義を集中させることで、運用と拡張を容易にする。

## 主要な型

- `AppMode`
  - `"normal"` または `"exam"` を表す列挙型。
  - 通常会話モードと試験モードの切り替えに使われる。

- `MessageRole`
  - `"user"` または `"assistant"` を表す列挙型。
  - 会話の発話者がユーザーか AI かを区別する。

- `Message`
  - `id: string` — メッセージの一意 ID。
  - `role: MessageRole` — 発話者。
  - `content: string` — メッセージ本文。
  - `timestamp: Date` — 送信時刻。localStorage への保存時は ISO 文字列にシリアライズし、読み込み時に `new Date()` で復元する。
  - `advice?: string` — ユーザー発話に対する評価アドバイス（Markdown 文字列）。EvalPopup で一度取得した内容をキャッシュし、再クリック時の API 呼び出しを省略する。`role === "assistant"` のメッセージには不使用。
  - 会話履歴や UI 表示、API 送信前の内部データとして使われる。

- `ExamResult`
  - `score: number` — 100 点満点のスコア。
  - `feedback: string` — 日本語のフィードバック。
  - Exam モード終了時に評価結果として保持する。

- `PersonaSettings`
  - `prompt: string` — AI に渡すシステムプロンプト／人格設定。
  - AI の応答スタイルをカスタマイズするために利用する。

- `ApiMessage`
  - `role: MessageRole` — AI API へ渡すメッセージの発話者。
  - `content: string` — AI API へ渡す本文。
  - `timestamp` などの内部フィールドを省き、外部 API 向けの形式を表現する。

- `Session`（推奨）
  - `id: string` — セッションの一意識別子。
  - `name?: string` — セッション名（任意）。
  - `mode: AppMode` — セッションが `normal` か `exam` かの種別。
  - `messages: Message[]` — 通常会話の履歴。
  - `examMessages: Message[]` — Exam モードの履歴。
  - `examRallyCount: number` — Exam 用ラリー数。
  - `examResult?: ExamResult | null` — 試験結果（存在する場合）。
  - `createdAt: string` / `updatedAt?: string` — メタ情報（ISO 日時文字列推奨）。

Session 型を `types/index.ts` に追加しておくと、コンテキストやセッション永続化の実装が楽になります。

## 要件

- `Message` は UI での表示と内部状態の根幹をなすため、必要なフィールドを常に持つこと。
- `ApiMessage` は外部 API との連携に使う軽量な形式であること。
- `AppMode` と `MessageRole` は文字列リテラル型として定義し、型安全にモード・発話者を扱えるようにする。
- 日付や数値の型は厳密に定義し、`string` による曖昧なデータ扱いを避ける。

## 拡張の考え方

- 将来的に会話履歴にメタデータ（添付ファイル、翻訳結果、ステータス等）を追加する場合は、`Message` のフィールドを拡張する。
- `ExamResult` に `passed: boolean` や `criteria` を追加して評価基準を細かく表現することも可能。
- `PersonaSettings` に `language` や `tone` などを追加して、AI の設定を多次元に制御できるようにする。

## 使い方

`types/index.ts` を `import type` で参照し、コンポーネント・Context・API で型を共有します。

```ts
import type { Message, AppMode, PersonaSettings } from '@/types';
```

以上が `types/` フォルダの設計と要件です。
